# 實作 AspNetCore Grpc 專案

參考很多網路上文章，比較少這方面的資料，我所找到的都列在最下方的參考連結區，不過有些專案跑起來會失敗。花了很多時間查問題與了解，底下紀錄過程。

# 環境相關

  • windows10 OS
  • Serve 端為 .Net Core 3.0
  • Client 端為 Vuejs 2.6.10
  • Docker 19.03

# Server

NET Core API 專案

  • 建立新專案
  • 選擇 ASP.NET Core Web 應用程式
  • 輸入專案名稱與路徑
  • 上方下拉選單為 .NET Core ASP.NET Core 3.0。
  • 範本選擇 API，右邊勾選設定 https 與啟用 Docker，下拉選擇 Linux
  • 點選建立
  • 直接執行 Ctrl+F5 專案應該可以正常執行

到這裡已把 api 專案建立起來，會發現在 .net Core 3.0 範本預設 api 回傳內容為天氣的報告。

TIP

.net core 3.0 提供了一個 gRPC 範本，可以快速產生一個 grpc 專案 參考。 但這次實作是用 .net core API 範本改成 grpc 的方式。

  • 安裝 grpc-dotnet 套件(可使用 nuget 安裝)套件

    PM> Install-Package Grpc.AspNetCore -Version 2.24.0

  • 修改 Startup.cs 在 ConfigureServices 加入

     services.AddGrpc();

    在 Configure 的 UseEndpoints 將 API 註冊

     endpoints.MapGrpcService<WeatherForecastController>();

  • 新增資料夾 Protos，用來放置 proto 檔

  • 在 Protos 資料夾點選右鍵，選擇新增項目

  • 範本請選擇『通訊協定緩衝區檔案』，名稱輸入『weather』

  • 開始撰寫 Protocol Buffers 檔參考

// 使用 proto3 版本語法
syntax = "proto3";

option csharp_namespace = "grpcAPI.protos";

package Weather;

// 定義 Server grpc 定義，無論如何一定要給 request
service WeatherForecast {
  rpc WeatherForecast (WeatherRequest) returns (WeatherReply);
}

// 定義 request 格式，若無則可以空白
message WeatherRequest {
  
}
// 定義回傳類型
message MapFieldEntry {
 string Date = 1;
 string TemperatureC = 2;
 string Summary = 3;
}
// 定義回傳內容
message WeatherReply {
  repeated MapFieldEntry result = 1;
}
  • 在專案上點選右鍵，選取『編輯專案檔』，把剛才新增的 weather.proto 檔案加入，這樣 Grpc 才會編譯 proto 檔案。
<ItemGroup>
  <Protobuf Include="Protos\weather.proto" GrpcServices="Server" />
</ItemGroup>

  • 改寫 API，把 [ApiController] 與 [Route("[controller]")] 標籤移除, class 改為繼承 protos.WeatherForecast.WeatherForecastBase，並且 override proto 所產生的方法，並且加入 API 程式碼

      public class WeatherForecastController : protos.WeatherForecast.WeatherForecastBase
  {
   ......
   public override async Task<protos.WeatherReply> WeatherForecast(protos.WeatherRequest request, ServerCallContext context)
      {
          var rng = new Random();
          var data = Enumerable.Range(1, 5).Select(index => new protos.MapFieldEntry
          {
              Date = DateTime.Now.AddDays(index).ToString(),
              TemperatureC = rng.Next(-20, 55).ToString(),
              Summary = Summaries[rng.Next(Summaries.Length)]
          })
          .ToList();
          var result = new protos.WeatherReply();
          result.Result.AddRange(data);
          return await Task.FromResult(result);
      }
  }

# 測試用 Client

  • 建立新專案
  • 選擇『主控台應用程式』專案，名稱輸入：GrpcClient1
  • 安裝套件
    • Install-Package Grpc.Net.Client
    • Install-Package Google.Protobuf
    • Install-Package Grpc.Tools
  • 將 grpc server 裡的 Protos 資料夾整個複製過來
  • 撰寫測試 Code
  static  void Main(string[] args)
  {
    var channel = GrpcChannel.ForAddress("https://localhost:5001");
    var client = new WeatherForecast.WeatherForecastClient(channel);
    var reply = client.WeatherForecast( new WeatherRequest());

    Console.WriteLine("Weather: " + reply.Result);
            Console.WriteLine("Press any key to exit...");
            Console.ReadKey();
  }

先執行 Grpc Server 專案起來，在執行 Grpc Client 專案，看到如下畫面代表 Client 正確呼叫 server 並回傳天氣資料。
