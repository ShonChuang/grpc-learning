# grpc 主要優點

  • 現代化、高效能、輕量的 RPC 架構。
  • 根據預設使用 Protocol Buffers 的合約優先式 API 開發，使您得以進行不限於語言的實作。
  • 適用於多種語言的工具，可產生強型別伺服器及用戶端。
  • 支援用戶端、伺服器及雙向資料流呼叫。
  • 透過 Protobuf 二進位序列化減少網路使用量。

TIP

Azure App Service 或 IIS 目前不支援ASP.NET Core gRPC 。 Http.sys 的 HTTP/2 執行不支援 gRPC 所依賴的 HTTP 回應尾端標頭

TIP

HTTP/2

gRPC 需要 HTTP/2。 ASP.NET Core 的 gRPC 會驗證HttpRequest。通訊協定為HTTP/2。

Kestrel 支援大多數新式作業系統上的HTTP/2 。 根據預設，Kestrel 端點會設定為支援 HTTP/1.1 和 HTTP/2 連接。
